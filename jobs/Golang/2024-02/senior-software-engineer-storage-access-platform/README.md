# About the Team

The mission of the Storage Access Platform team is to make online data interactions simple, reliable and performant for the DoorDash Engineering teams. We help product teams move fast by building clean, robust data APIs so that our engineers can focus on building business logic without being exposed to data access complexities. This is a flexible/hybrid role and the candidate must be located in San Francisco, Sunnyvale, Seattle, or New York.

# About the Role

As a Software Engineer, you'll be working on building services, libraries on top of our online Databases. At DoorDash, we use Cassandra for Key Value needs, CRDB and Aurora PostgreSQL for transactional use cases, Redis for Caching the data in memory. As a platform team, we build and maintain several services including a Data gateway service to let teams talk to Databases using gRPC with many features added in the layer, a service API cache that can cache the API calls. We support PgBouncer as a proxy for Aurora PostgreSQL.

You will use your skills and experience in guiding the engineers and the management leadership on the right technical choices, mentor engineers and hold a high bar of technical competency.
